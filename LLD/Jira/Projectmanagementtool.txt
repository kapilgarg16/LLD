
What each class does — one line each

ProjectService: the organizer — creates projects, users, issues, assigns tasks, and returns lists.
Project: a named workspace that holds issues.
Issue: a task with title, description, status (TODO/IN_PROGRESS/DONE), assignee, timestamps and comments.
User: a person with id and name.
Comment: small message attached to an issue.


----------------
Classes — ultra-brief (attributes → main functions → memorise line)

User
    Attributes: id, name.
    Main funcs: constructor, getId(), getName()
    Line: “User = id + name.”

Project
    Attributes: id, name, description.
    Main funcs: constructor, getters.
    Line: “Project = named workspace.”

Issue
    Attributes: id, projectId, title, description, status, assigneeUserId, createdAt, updatedAt, comments.
    Main funcs: setters (setTitle, setDescription, setStatus, setAssignee), addComment(),
     getters. touch() updates updatedAt.
    Line: “Issue = task + status + assignee + comments.”

Comment

    Attributes: id, issueId, userId, text, createdAt.
    Main funcs: constructor, getters.
    Line: “Comment = who said what when.”

ProjectService

    Attributes: maps for users, projects, issues, plus projectIssues (project→issueIds) and
    userAssignedIssues (user→issueIds); simple counters for ids.

    Main funcs: createUser, createProject, createIssue, assignIssue, addComment,
    listIssuesByProject, listIssuesByUser, listIssuesByStatus.

    Line: “ProjectService = the organizer (creates/assigns/lists).”
